# link.acttub.com

인스타그램 프로필 바이오에 거는 **링크 허브** 한 장짜리 정적 페이지.

배포: <https://link.acttub.com> (Vercel 프로젝트 `link-hub`)

## 왜 있나

인스타 바이오는 링크 슬롯이 하나뿐인데, 게시물마다 목적지가 다르다.
인스타 기본 다중 링크 기능은 제목 30자만 보이는 선택 시트를 띄워서
클릭이 한 번 더 늘고, 어느 게시물에서 왔는지와 무관하게 같은 목록이 나온다.

그래서 바이오 링크는 이 페이지 하나로 두고, 여기서 목적지를 나눈다.

**단, 광고·스토리 링크는 이 페이지를 거치지 않는다.**
의도가 이미 정해진 트래픽이므로 각 사이트로 직행시킨다.
허브는 "어디서 왔는지 모르는" 프로필 방문자만 받는다.

## 구조 — 계층이 핵심

평평한 Linktree식 나열이 아니다. voice·acti는 acttub과 경쟁하는
형제 서비스가 아니라 코어로 태우려고 만든 유입 경로이므로,
같은 크기로 나열하면 깔때기가 옆으로 눕는다.

- **acttub** — 큰 메인 카드. 시선이 먼저 간다
- **가볍게 시작해보기** — 보이스 컬러(30초) · 연기 스타일 유형 찾기(2분). 작은 행

## 파일

```
index.html      허브 페이지 전부 (인라인 CSS, 빌드 없음)
go.html         채널 유입 기록용 경유 페이지 (아래 "채널별 유입 기록")
apps-script.gs  go.html이 보낸 기록을 구글 시트에 쌓는 수신 코드
vercel.json     cleanUrls — /go.html 을 /go 로도 열리게 한다
logo.png        볼트 브랜드 폴더의 로고를 180px로 축소한 것
```

의존성·빌드 단계 없음. `index.html` 열면 그대로 보인다.

## 채널별 유입 기록 (`/go`)

`acttub.com`은 소스 저장소가 특정되지 않아 우리가 계측을 붙일 수 없다.
그래서 "필름메이커스 글을 보고 왔다"를 남기려면 **우리가 소유한 이 도메인을 한 번 거쳐야** 한다.

```
https://link.acttub.com/go?from=filmmakers
        → from 기록 → https://acttub.com/?from=filmmakers 로 자동 이동
```

- 사람에게는 거의 직행처럼 보인다(화면이 뜨기 전에 넘어간다).
- 시트에 남는 것: 도착 시각 · 채널(`from`) · 전체 링크 파라미터 · **출처 사이트**(도메인까지만) · 클릭 ID.
  - 직전 페이지의 전체 주소는 일부러 버린다 — 보내는 쪽 설정에 따라 검색어·계정 ID가 딸려올 수 있다.
  - 클릭 ID는 클릭 1건마다 새로 만든다. **같은 사람의 재방문을 잇지 않는다** —
    그러려면 브라우저에 식별자를 남겨야 하는데 채널을 세는 데는 필요 없다.
- 원래 쿼리는 목적지까지 그대로 넘긴다 — 나중에 `acttub.com`에 계측이 붙으면 거기서도 보인다.
- 뒤로가기를 누르면 원래 보던 글로 돌아간다(`location.replace`라 경유 페이지가 기록에 안 남는다).

### 켜는 법

`go.html`의 `ENDPOINT`가 **비어 있으면 기록만 안 되고 이동은 정상 동작한다.**
채우는 순서는 `apps-script.gs` 맨 위에 있다.

⚠️ **리뷰 폼과 시트·배포 URL을 절대 공유하지 않는다.** 이 저장소는 공개라 여기 적힌
주소는 누구나 볼 수 있고, 그건 곧 누구나 POST를 보낼 수 있다는 뜻이다. 이 시트는 가짜 행이
쌓이는 정도가 최악이라 감수하지만, 전화번호가 들어 있는 검증 응답 시트에는 같은 문을 열면 안 된다.

새 채널을 추가할 때 코드는 건드리지 않는다. `?from=` 값만 바꿔 링크를 만들면 된다.

## 유의사항

- **`target="_blank"` 금지.** 인스타 인앱 브라우저에서 새 탭이 안 열리거나
  웹뷰가 겹쳐 사용자가 돌아갈 길을 잃는다. 전부 같은 탭 이동.
- **링크에 UTM 유지.** `utm_source=linkhub&utm_medium=bio&utm_campaign=hub`.
  허브에서 셋 중 어디로 갈리는지 이걸로 측정한다.
- **문구는 제품 가드레일 적용 대상.** 점수·등급·평가·처방 프레임 금지.
  볼트 `AGENTS.md`의 "제품 가드레일" 절이 정본. 문구 고치면 다시 검사할 것.
- 로고는 볼트의 브랜드 폴더가 정본. 바꿀 일이 있으면 거기서 가져와 축소한다.

## 배포

`main`에 push하면 Vercel이 자동으로 프로덕션(`link.acttub.com`)에 반영한다.
로컬에서 `vercel --prod`를 치지 않는다.

```bash
git push origin main
```

다른 브랜치로 push하면 미리보기 URL만 생기고 프로덕션은 그대로다.

DNS는 Cloudflare에 CNAME `link` → Vercel, **proxy는 반드시 DNS only(회색 구름)**.
주황 구름이면 인증서 발급이 막힌다.
