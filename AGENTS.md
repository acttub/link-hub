# link-hub — 작업 지침

`link.acttub.com` (인스타 바이오용 링크 허브) 저장소.
**무엇이고 왜 이런 구조인지는 `README.md`를 먼저 읽는다.** 여기엔 작업 규칙만 둔다.

## 소유·정본

- **최우영 단독 소유** 서브프로젝트. 팀 합의를 묻지 않고 바로 고쳐도 된다.
- **GitHub(`acttub/link-hub`)가 정본**이고 로컬 폴더는 작업본이다. 고친 뒤 push해야 배포에 반영된다.
- 제품 가드레일 정본은 볼트 `/Users/coo/Soma/AGENTS.md`의 "제품 가드레일" 절이다. 여기에 목록을 복제하지 않는다.

## 고칠 때 지킬 것

- **사용자 노출 문구를 고치면 가드레일을 다시 검사한다.** 점수·등급·평가·판정·강점약점·처방 프레임 금지. "테스트"처럼 판정으로 읽힐 단어도 주의.
- **`target="_blank"`를 새로 넣지 않는다.** 인스타 인앱 브라우저에서 깨진다.
- **링크에서 UTM을 떼지 않는다.** 이게 없으면 허브가 실제로 작동하는지 알 수 없다.
- **계층을 평평하게 만들지 않는다.** acttub이 크고, voice·acti는 그 아래 작은 행이다. 같은 크기로 나열하면 깔때기가 무너진다.
- 빌드 단계를 도입하지 않는다. 정적 HTML 한 장으로 유지한다.

## 링크를 추가·교체할 때

새 서브프로젝트가 생겨서 행을 추가한다면:
- "가볍게 시작해보기" 묶음에만 넣는다. 메인 카드는 acttub 코어 자리다.
- 소요시간 배지(예: `30초`, `2분`)를 붙인다 — 부담 없음을 먼저 보이는 게 목적이다.
- UTM을 동일 형식으로 붙인다.
- 목적지가 실제로 200을 반환하는지 확인하고 커밋한다.

## 배포

`vercel deploy --prod`. DNS·인증서 관련 주의사항은 README 참조.
