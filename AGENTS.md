# link-hub — 작업 지침

`link.acttub.com` (인스타 바이오용 링크 허브) 저장소.
**무엇이고 왜 이런 구조인지는 `README.md`를 먼저 읽는다.** 여기엔 작업 규칙만 둔다.

## 소유·정본

- **최우영 단독 소유** 서브프로젝트. 팀 합의를 묻지 않고 바로 고쳐도 된다.
- **GitHub(`acttub/link-hub`)가 정본**이고 로컬 폴더는 작업본이다. 고친 뒤 push해야 배포에 반영된다.
- 제품 가드레일 정본은 볼트 루트의 `AGENTS.md` "제품 가드레일" 절이다. 여기에 목록을 복제하지 않는다.

## 고칠 때 지킬 것

- **사용자 노출 문구를 고치면 가드레일을 다시 검사한다.** 점수·등급·평가·판정·강점약점·처방 프레임 금지. "테스트"처럼 판정으로 읽힐 단어도 주의.
- **`target="_blank"`를 새로 넣지 않는다.** 인스타 인앱 브라우저에서 깨진다.
- **링크에서 UTM을 떼지 않는다.** 이게 없으면 허브가 실제로 작동하는지 알 수 없다.
- **계층을 평평하게 만들지 않는다.** acttub이 크고, voice·acti는 그 아래 작은 행이다. 같은 크기로 나열하면 깔때기가 무너진다.
- 페이지에 빌드 단계를 도입하지 않는다. `index.html`은 손으로 고치는 정적 한 장으로 유지한다.

## ⚠️ 문구를 고치면 폰트를 다시 만든다

`fonts/*.woff2`는 **`index.html`에 실제로 쓰인 글자만** 담은 서브셋이다(3굵기 합계 36KB).
전체 한글을 실으면 굵기당 1.5MB이라 이렇게 한 것이다.

그래서 **문구에 새 글자가 들어가면 그 글자는 시스템 폰트로 렌더돼 서체가 섞인다.**
문구를 고쳤으면 반드시:

```bash
uv run --with "fonttools[woff]" python build-fonts.py
```

원본 otf는 볼트 `brand/fonts/`에 있다(공개 저장소에 커밋하지 않는다).
서브셋에는 영문·숫자·기본 문장부호가 안전망으로 항상 포함되므로, 한글을 새로 쓸 때만 문제가 된다.

## 측정

페이지 조회수는 Vercel Web Analytics(쿠키리스)로 본다.

**링크별 클릭은 이 페이지에서 셀 수 없다.** Vercel의 커스텀 이벤트(`va('event', …)`)는
Pro 플랜 전용이고 이 계정은 Hobby다. 코드를 넣어도 집계되지 않으니 시도하지 않는다.

대신 **목적지 쪽에서 역으로 센다.** `voice`·`acti`의 `index.html`에도 같은
`/_vercel/insights/script.js`를 넣어두었다. 각 목적지 대시보드의 **Referrers**에서
`link.acttub.com` 유입 수를 보고, 허브 조회수로 나누면 링크별 CTR이 나온다.

- `acttub.com`에는 우리가 계측을 못 붙인다(소스 저장소 미특정). 그래서 **코어 카드는 `/go`를 거친다**
  — `/go?from=hub-core`. 이 클릭은 Vercel이 아니라 **구글 시트**에 쌓인다(README "채널별 유입 기록").
  옛 "코어 링크는 측정 공백" 서술은 이걸로 해소됐다.
- **voice·acti 카드는 직행을 유지한다.** 목적지에 우리 계측이 있어 referrer로 이미 세지므로,
  `/go`를 끼우면 이미 되는 걸 건드리면서 홉만 늘린다.
- 그래서 숫자가 **두 곳에 나뉜다**: 코어는 시트, voice·acti는 Vercel. 합칠 때 헷갈리지 않게
  `from=hub-*`(허브 안에서 누른 것)과 `from=<채널>`(외부에서 들어온 것)을 구분해 읽는다.
- UTM 파라미터로 거르는 기능은 Web Analytics Plus부터다. 지금 UTM은 목적지가
  자체적으로 읽을 때를 위한 것이니 **떼지 않는다.**
- Hobby 한도: 이벤트 5만/월(계정 전체 합산), 데이터 보관 1개월.

숫자는 대시보드 대신 조회 API로도 뽑을 수 있다
(`GET /v1/query/web-analytics/visits/aggregate`, `by=referrerHostname`).

### 켤 때 걸렸던 함정 두 개 (2026-07-20)

1. **대시보드에서 Enable을 누른 뒤 재배포해야 라우트가 생긴다.** 순서를 지키지
   않으면 영원히 404다. REST API에도 CLI에도 켜는 명령이 없다 —
   프로젝트 API의 `webAnalytics.id` 필드는 활성화의 증거가 아니다.
2. **SPA 폴백 리라이트가 스크립트를 삼킬 수 있다.** acti가 그랬다.
   이때는 404가 아니라 **200 text/html**이 와서 정상처럼 보인다.

그래서 확인은 상태 코드가 아니라 **content-type**으로 한다.

```bash
curl -s -o /dev/null -w "%{http_code} %{content_type}\n" \
  https://link.acttub.com/_vercel/insights/script.js   # 200 application/javascript 여야 한다
```

## 링크를 추가·교체할 때

새 서브프로젝트가 생겨서 행을 추가한다면:
- "가볍게 시작해보기" 묶음에만 넣는다. 메인 카드는 acttub 코어 자리다.
- 소요시간 배지(예: `30초`, `2분`)를 붙인다 — 부담 없음을 먼저 보이는 게 목적이다.
- UTM을 동일 형식으로 붙인다.
- 목적지가 실제로 200을 반환하는지 확인하고 커밋한다.

## 배포

`git push origin main` 하면 자동 배포된다. 로컬에서 `vercel --prod`를 치지 않는다.
DNS·인증서 주의사항은 README 참조.
