# link-hub — 작업 지침

`link.acttub.com` (인스타 바이오용 링크 허브) 저장소.
**무엇이고 왜 이런 구조인지는 `README.md`를 먼저 읽는다.** 여기엔 작업 규칙만 둔다.

## 소유·정본

- **최우영 단독 소유** 서브프로젝트. 팀 합의를 묻지 않고 바로 고쳐도 된다.
- **GitHub(`acttub/link-hub`)가 정본**이고 로컬 폴더는 작업본이다. 고친 뒤 push해야 배포에 반영된다.
- 제품 가드레일 정본은 볼트 루트의 `AGENTS.md` "제품 가드레일" 절이다. 여기에 목록을 복제하지 않는다.

## 고칠 때 지킬 것

- **사용자 노출 문구를 고치면 가드레일을 다시 검사한다.** 점수·등급·평가·판정·강점약점·처방 프레임 금지. "테스트"처럼 판정으로 읽힐 단어도 주의.
- **`target="_blank"`를 새로 넣지 않는다.** 인스타 인앱 브라우저에서 깨진다.
- **링크에서 UTM을 떼지 않는다.** 이게 없으면 허브가 실제로 작동하는지 알 수 없다.
- **계층을 평평하게 만들지 않는다.** acttub이 크고, voice·acti는 그 아래 작은 행이다. 같은 크기로 나열하면 깔때기가 무너진다.
- 페이지에 빌드 단계를 도입하지 않는다. `index.html`은 손으로 고치는 정적 한 장으로 유지한다.

## ⚠️ 문구를 고치면 폰트를 다시 만든다

`fonts/*.woff2`는 **`index.html`에 실제로 쓰인 글자만** 담은 서브셋이다(3굵기 합계 36KB).
전체 한글을 실으면 굵기당 1.5MB이라 이렇게 한 것이다.

그래서 **문구에 새 글자가 들어가면 그 글자는 시스템 폰트로 렌더돼 서체가 섞인다.**
문구를 고쳤으면 반드시:

```bash
uv run --with "fonttools[woff]" python build-fonts.py
```

원본 otf는 볼트 `brand/fonts/`에 있다(공개 저장소에 커밋하지 않는다).
서브셋에는 영문·숫자·기본 문장부호가 안전망으로 항상 포함되므로, 한글을 새로 쓸 때만 문제가 된다.

## 측정

페이지 조회수는 Vercel Web Analytics(쿠키리스)로 본다.
**링크별 클릭은 이 페이지에서 세지 않는다** — 목적지 3곳이 각자 분석 도구로
`utm_source=linkhub` 유입을 집계하고, 그 합을 조회수로 나눠 CTR을 구한다.
그래서 **링크에서 UTM을 떼면 측정이 통째로 불가능해진다.**

## 링크를 추가·교체할 때

새 서브프로젝트가 생겨서 행을 추가한다면:
- "가볍게 시작해보기" 묶음에만 넣는다. 메인 카드는 acttub 코어 자리다.
- 소요시간 배지(예: `30초`, `2분`)를 붙인다 — 부담 없음을 먼저 보이는 게 목적이다.
- UTM을 동일 형식으로 붙인다.
- 목적지가 실제로 200을 반환하는지 확인하고 커밋한다.

## 배포

`git push origin main` 하면 자동 배포된다. 로컬에서 `vercel --prod`를 치지 않는다.
DNS·인증서 주의사항은 README 참조.
